# Poor Charlie's Almanack Skill

すぐに使える AI スキルパック——チャーリー・マンガーのコア思考法を、12個の個別起動・組み合わせ可能な判断モジュールに蒸留しました。

これは本の要約ページではありません。本はこれらのスキルの源泉であり、主役ではありません。

## このスキル群が解決する問題

- 重要な決断を直感だけで下したくない
- 認知バイアス、インセンティブの歪み、判断の盲点を特定したい
- 「逆転思考」「能力の圏」「多元思考モデル」を呼び出し可能なツールにしたい
- 投資・ビジネス分析に使える再利用可能な思考フレームワークが必要
- チームの意思決定に構造化されたチェックを導入したい

## 何が蒸留されたか

『貧しきチャーリーの格言』から12個の三重検証済みスキルを抽出し、4つの次元をカバーしています：

- **コア思考法**：`inversion-thinking`（逆転思考）、`latticework-mental-models`（多元思考モデルグリッド）、`darwinian-objectivity`（ダーウィン的客観性）、`circle-of-competence`（能力の圏）
- **認知バイアス防御**：`dual-track-analysis`（デュアルトラック分析）、`misjudgment-psychology-checklist`（誤判心理学チェックリスト）、`lollapalooza-effect`（ラパルーザ効果）、`incentive-analysis`（インセンティブ分析）、`deprival-superreaction`（喪失過剰反応）
- **投資・ビジネス分析**：`value-assessment`（価値評価）、`scale-advantage-analysis`（規模優位性分析）
- **統合実行フレームワーク**：`checklist-decision`（チェックリスト意思決定）

抽出プロセスは49個の候補から開始し、合格率は24.5%——4つの候補のうち1つだけがスキルになりました。

## 効果例

### 例1：投資判断における自己疑念

**ユーザーの質問**

「SaaS企業への投資を検討しています。成長は速いですが、創業者のプレゼンに乗せられているだけかもしれません。どう確認すればいいですか？」

**スキルの判断**

- まず `circle-of-competence` を起動——この企業は本当に理解できる範囲内か？
- 次に `darwinian-objectivity` を起動——反証を強制的に探す
- 同時に `incentive-analysis` を呼び出し——創業者、投資家、営業チームのインセンティブが一致しているか確認
- 最後に `checklist-decision` で——判断を検証可能なチェックリストに集約

**回答例**

> まず「良い会社か」ではなく「本当に理解できる会社か」を判断してください。4つのことを答える必要があります：なぜ成長し続けられるのか、なぜ顧客が簡単に離脱しないのか、今後2〜3年で成長を妨げる最も可能性の高い要因は何か、創業者がいなくても会社は機能するのか。この4つのうち2つが明確に答えられない場合、その会社はおそらくあなたの能力の圏外です。次に逆転して考えてみてください：どんな条件下でこの会社は失敗するか？最も一般的な3つの答えは、顧客獲得コストの急増、製品の代替可能性の高さ、チームのインセンティブの不一致です。これらをチェックリストにまとめてから、調査を続けるかどうかを決めてください。

### 例2：規模優位性の罠

**ユーザーの質問**

「業界リーダーがどんどん大きくなっていますが、それは自動的に優位性を意味するのでしょうか？」

**スキルの判断**

- `scale-advantage-analysis` を起動
- 極端な結果が関係する場合は `lollapalooza-effect` を追加
- 最終的に投資判断に至る場合は `value-assessment` に接続

**回答例**

> 規模は自動的に優位性とイコールではありません。まず規模がもたらすものを見てください：調達コストの低下、ブランド認知の強化、ネットワーク効果の増大、情報優位性の拡大。次に規模がコストを増やし始めているかを見てください：組織の肥大化、意思決定の遅延、インセンティブの歪み、社内政治の増加。本当に重要なのは「大きいか」ではなく、規模の優位性がまだ増幅しているのか、すでに官僚的コストに食われているのかです。両方の力が同時に存在する場合、市場リーダーだからといって「強い」とデフォルトで仮定することはできません。

## これらのスキルはどのように生成されたか

これらのスキルは **cangjie-skill** の RIA-TV++ パイプラインを使用して生成されました。6段階のプロセスです：

1. **全書理解（Adler分析）** — モーティマー・アドラーの分析的読書法で、全書を構造・解釈・批判・応用の4ステップで分解し、`BOOK_OVERVIEW.md` を生成
2. **並行抽出** — 5つの専門エクストラクター（フレームワーク、原則、事例、反例、用語）が同時に実行され、原文から候補ユニットを抽出
3. **三重検証** — 各候補は3つのチェックを通過する必要があります：書中に少なくとも2つの独立した裏付けがあるか（クロスドメイン）、書中に明示されていない新しい質問に答えられるか（予測力）、常識ではないか（独自性）。合格率は通常25〜50%
4. **RIA++ 構築** — 検証済みの内容を6つの次元に構造化：R（原文引用）/ I（自分の言葉での再構築）/ A1（書中の事例）/ A2（将来のトリガーシーン）/ E（実行可能ステップ）/ B（境界と盲点）
5. **ツェッテルカステン連携** — スキル間の依存、対比、構成関係を特定し、参照グラフ付きの `INDEX.md` を生成
6. **ストレステスト** — 各スキルに囮問題を含むテストプロンプトを設計。不合格は段階4に戻って全面的に再構築

各 `SKILL.md` はこのプロセスの実際の出力です。`candidates/` と `rejected/` ディレクトリは完全な監査証跡として保持されています。
